WIDGET FLOTANTE (van a wp-content/uploads/2026/07/)
─────────────────────────────────────────────────────
powerups-edge-frame.html
    Documento del iframe. Carga el CSS, el marcado del chat y el JS del widget.
    Es el único HTML que el iframe renderiza. Recibe la config via query string.

powerups-edge-widget.js
    Lógica completa del chat: CONFIG, llamadas a la API, ECharts inline, barra
    de uso diaria (X/15), paywall, resize, postMessage al host, localStorage.
    NO incluye formulario de email (eso vive en premium-portal.html).

powerups-edge-widget.css
    Todos los estilos del panel, burbuja, composer y estados del widget.

powerups-edge-widget-markup.html
    Fragmento HTML de referencia. Debe mantenerse sincronizado con el bloque
    <body> de powerups-edge-frame.html. No se sube solo a producción.

LOADERS (solo uno activo en producción)
─────────────────────────────────────────────────────
bebuilder-install-snippet.html  ← LOADER RECOMENDADO (inline)
    Plantilla de instalación. Copia su bloque <script>…</script> a
    Theme Options → Hooks → Bottom. NO subir este .html a Medios.
    Incluye POWERUPS_WIDGET_CONFIG + LOADER con assetsBase 2026/07/,
    frameUrl, API_BASE, CHECKOUT_URL, PREMIUM_PORTAL_URL, DASHBOARD_SESSION_URL.
    También carga powerups-bold-checkout.js cuando existe #pu-bold-mount.

widget-loader.js  ← loader alternativo como archivo externo
    Mismo comportamiento que el inline.
    Expone window.POWERUPS_WIDGET_LOADER_API:
      · abrirInsightFlow() / cerrarInsightFlow()

PÁGINAS (WordPress → pegar HTML)
─────────────────────────────────────────────────────
premium-portal.html
    Portal "Mis tableros" + LOGIN POR CORREO (#portal-login):
      1. Usuario ingresa email → POST /api/v1/billing/link-email
      2. Si free → revela #pu-pro-card / #pu-bold-mount (botón Bold)
      3. Si premium → oculta pago y muestra plan ilimitado
    También: grid de gráficas desde localStorage, demo, acceso por token.
    URL: https://www.powerupsagencia.com/portal-premium/

premium-dashboard-session.html
    Visor ECharts a pantalla completa (token o fallback localStorage).
    URL: https://www.powerupsagencia.com/dashboard-premium-echarts/

ENDPOINTS DE API USADOS
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Widget (powerups-edge-widget.js):
  POST /api/v1/chat
  POST /api/v1/chat/upload
  GET  /api/v1/chat/:id/credits  (o billing/status según implementación)
  GET  /download/:token          (charts free; pdf/excel → 403 si free)

Portal (premium-portal.html):
  POST /api/v1/billing/link-email
  GET  /api/v1/billing/status?chat_id=
  GET  /api/v1/billing/bold-checkout?chat_id=  (vía powerups-bold-checkout.js)

Dashboard session:
  GET  /api/v1/dashboard/session/{token}
       → 200 { echarts_option }
       → 202 preparando / 404 → fallback localStorage
