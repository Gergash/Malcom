WIDGET FLOTANTE (van a wp-content/uploads/2026/07/)
─────────────────────────────────────────────────────
powerups-edge-frame.html
    Documento del iframe. Carga el CSS, el marcado del chat y el JS del widget.
    Es el único HTML que el iframe renderiza. Recibe la config via query string.

powerups-edge-widget.js  (v=7 en el <script> del frame)
    Lógica completa del chat: CONFIG, llamadas a la API, ECharts inline, barra
    de uso, paywall, resize, postMessage al host, localStorage.

powerups-edge-widget.css  (v=7 en el <link> del frame)
    Todos los estilos del panel, burbuja, composer y estados del widget.

powerups-edge-widget-markup.html
    Fragmento HTML de referencia. Debe mantenerse sincronizado con el bloque
    <body> de powerups-edge-frame.html. No se sube solo a producción.

LOADERS (solo uno activo en producción)
─────────────────────────────────────────────────────
bebuilder-install-snippet.html  ← LOADER RECOMENDADO (inline)
    Plantilla de instalación. Copia su bloque <script>…</script> a
    Theme Options → Hooks → Bottom. NO subir este .html a Medios;
    si lo abres en el navegador puede verse en blanco (es normal: es JS puro).
    Ventaja: no depende de cargar widget-loader.js por HTTP; evita 404 por
    caché agresiva en Medios.
    Incluye window.POWERUPS_WIDGET_CONFIG y window.POWERUPS_WIDGET_LOADER
    ya pre-configurados con assetsBase, frameUrl, API_BASE, CHECKOUT_URL,
    PREMIUM_PORTAL_URL y DASHBOARD_SESSION_URL.

widget-loader.js  ← loader alternativo como archivo externo
    Mismo comportamiento que el inline, pero se referencia con
    <script src=".../widget-loader.js" defer>.
    Detecta su propia URL (document.currentScript.src) para auto-resolver
    el baseDir y construir el frameUrl sin config manual.
    Expone window.POWERUPS_WIDGET_LOADER_API con:
      · abrirInsightFlow()  — abre el panel programáticamente
      · cerrarInsightFlow() — cierra el panel programáticamente

PÁGINAS PREMIUM (WordPress → Crear página → Pegar HTML como bloque "HTML personalizado" o en el editor de código)
─────────────────────────────────────────────────────
premium-portal.html
    Portal "Mis tableros". Lee desde localStorage las últimas gráficas ECharts
    generadas en el chat (clave: powerups_edge_charts_{chatId}) y las muestra
    en un grid con descarga PNG individual. Si no hay tableros muestra estado
    vacío + demo animada con datos de ejemplo. También tiene:
      · Sección "Qué incluye tu plan" (beneficios)
      · Campo de acceso por token (legacy / enlace externo compartido)
    Se actualiza automáticamente via evento storage cuando el widget genera
    una nueva gráfica mientras la página está abierta.
    URL de producción esperada: https://www.powerupsagencia.com/portal-premium/

premium-dashboard-session.html
    Visor de tablero individual en pantalla completa. Carga la gráfica ECharts
    desde el endpoint GET /api/v1/dashboard/session/{token}.
    Flujo:
      1. Lee ?token= de la URL.
      2. Fetch al API. Si 202 → "preparando", si 404 → fallback a localStorage.
      3. Cualquier fallo de red/CORS → fallback a gráficas locales del mismo
         chat (localStorage powerups_edge_charts_{chatId}).
      4. Modo local: navegación ← n/N → entre gráficas acumuladas.
    Acciones: Descargar PNG (resolución ×2), Recargar datos, enlace al Portal.
    URL de producción esperada:
      https://www.powerupsagencia.com/dashboard-premium-echarts/

ARQUITECTURA DEL WIDGET FLOTANTE
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

No necesitas un bloque <iframe> en el diseño de BeBuilder.
El loader (inline o widget-loader.js) crea un div#powerups-edge-widget-host
fijo en la esquina inferior derecha y monta el iframe desde JS.

Estados de tamaño del contenedor host (div fijo):
  · Cerrado (inicial):   336 × 96 px  — solo la burbuja con el botón
  · Abierto (chat):      min(960px, 100vw) × min(92vh, 100vh) — panel completo

Comunicación host ↔ iframe:
  · El widget envía postMessage { type: 'insightflow-widget', action: 'resize',
    open: true|false, width, height } cuando el usuario abre o cierra el panel.
  · El loader escucha ese mensaje y ajusta el div host en consecuencia.
  · Respaldo (mismo origen): MutationObserver sobre la clase .is-open del
    #powerups-edge-chat dentro del iframe, por si hay caché vieja del widget.js.

FLUJO DE CONFIGURACIÓN
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

WordPress (hook Bottom)
  └─ window.POWERUPS_WIDGET_CONFIG = { API_BASE, CHECKOUT_URL, … }
  └─ Loader (inline o widget-loader.js)
        └─ Construye src del iframe con query string:
             ?apiBase=…&checkoutUrl=…&premiumPortalUrl=…
             &dashboardSessionUrl=…&username=…
        └─ powerups-edge-frame.html (en Medios)
              └─ powerups-edge-widget.js
                    └─ Lee query string → Object.assign sobre DEFAULT_CONFIG
                    └─ Guarda API_BASE en localStorage['powerups_edge_api_base']
                       para que premium-dashboard-session.html lo recupere

window.POWERUPS_WIDGET_CONFIG → NO llega automáticamente al interior del iframe
cuando el frame está en otro origen (uploads vs. dominio WP). El loader lo copia
explícitamente como query string del src.

CLAVES DE localStorage (origen del iframe)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

powerups_edge_chat_id_v1          → chat_id del usuario
powerups_edge_history_v1          → historial de mensajes
powerups_edge_charts_{chatId}     → array de opciones ECharts (max 6, LIFO)
powerups_edge_dash_tok_{chatId}   → token del último tablero de la sesión
powerups_edge_panel_size_v1       → tamaño redimensionado del panel (usuario)
powerups_edge_api_base            → API_BASE para que el visor pueda recuperarla

IMPORTANTE: el localStorage pertenece al ORIGEN de powerups-edge-frame.html
(wp-content/uploads/…). Las páginas premium y el visor leen del mismo origen
solo si también están servidas desde ese dominio. Si el frame migra de
CDN/subdominio hay que actualizar las URLs de portal y dashboard.

ENDPOINTS DE API USADOS
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

GET  /api/v1/dashboard/session/{token}
     → Devuelve { echarts_option: { … } }  (200 OK)
     → 202 Accepted si el tablero aún se está generando
     → 404 si no existe sesión (el visor hace fallback a localStorage)
     → 401/403/409 → el visor manda postMessage token_refresh al padre

Todos los demás endpoints (chat, upload, quota, PDF, Excel) los maneja
powerups-edge-widget.js. Ver DEFAULT_CONFIG en ese archivo para el API_BASE.
